# AI — аудит

## 1. Текущее состояние

ИИ-чат через Claude API. Контекст: холодильник пользователя + история сообщений.

**Лимиты:**
- Гость: 2 сообщения/день по IP (in-memory счётчик в `chat.js`)
- Авторизованный пользователь: 10 сообщений/день (через БД, `ChatMessage` модель)
- PRO/ADMIN: без ограничений

**Блокировка нерелевантных запросов:** `messageFilter.js` — blocklist по ключевым словам.

**Ответы с блюдами:** ИИ возвращает маркеры `[DISH:id]`, фронт подставляет inline-карточки блюд.

**Telegram:** ИИ-чат временно скрыт (кнопка убрана из меню, код закомментирован в `bot.js`).

## 2. Найденные проблемы

| Проблема | Приоритет |
|---|---|
| Гостевой счётчик in-memory — сбрасывается при рестарте PM2 | medium |
| Blocklist-фильтр даёт много ложных срабатываний | medium |
| Нет логирования стоимости токенов | low |

## 3. Решения

- In-memory счётчик для гостей — приемлемо на текущем этапе (сброс раз в сутки не критичен)
- Blocklist — пока оставить, переработка на whitelist в бэклоге
- Логирование токенов — в бэклоге (анализ API-затрат)

## 4. Реализация

**Файлы:**
- `backend/src/routes/chat.js` — основная логика, лимиты, контекст
- `backend/src/lib/messageFilter.js` — blocklist-фильтр

**Prompt:** системный промпт включает список ингредиентов из холодильника пользователя.

## 5. Открытые вопросы

- Переработка фильтра: blocklist → whitelist (блокировать если нет food-related ключевых слов)
- Анализ API-затрат: стоимость на запрос, прогноз при росте пользователей
- Включение ИИ-чата в Telegram-боте (временно скрыт)

## 6. Договорённости

- Гостевой лимит: 2 сообщения/день — не увеличивать без бизнес-решения
- Пользовательский лимит: 10/день — аналогично
- `messageFilter.js` — не удалять, только дорабатывать
- Маркеры `[DISH:id]` в ответе ИИ — не менять формат без синхронного обновления парсера на фронте
