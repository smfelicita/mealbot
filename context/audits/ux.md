# UX — аудит

## 1. Текущее состояние

Светлая тема, минималистичный дизайн. Фон `#F6F4EF`.

**Цветовые токены:**
| Токен | Назначение |
|---|---|
| `bg` | Основной фон (`#F6F4EF`) |
| `bg-2` | Фон карточек, блоков |
| `bg-3` | Фон инпутов, вторичных областей |
| `text` | Основной текст |
| `text-2` | Вторичный текст (подписи, описания) |
| `text-3` | Третичный (плейсхолдеры, метки) |
| `border` | Границы элементов |
| `accent` | Акцентный цвет (`#C4704A`) |
| `sage` | Зелёный акцент (`#5C7A59`) |

**Тени:**
- `shadow-card` — карточки блюд
- `shadow-sm` — лёгкие тени (теги, кнопки)
- `shadow-md` — модалки, меню
- `shadow-accent` — кнопки с акцентом
- `shadow-sage` — кнопки sage
- `shadow-top` — нижние оверлеи (action menu)

**Spacing:** gap-3 между карточками, px-4/px-5 для контейнеров, pt-4/pb-8 для content-секций.

**Состояния экранов:** каждый экран имеет: default, loading (Loader/Skeleton), empty (EmptyState), error (Toast).

## 2. Найденные проблемы

| Проблема | Приоритет |
|---|---|
| Inline styles с цветами вместо токенов (color, background, boxShadow) | high |
| `sage` в tailwind.config = `#4A6B47`, в коде везде `#5C7A59` — рассинхрон | high |
| Отсутствовали shadow-токены, пришлось писать inline | medium |
| Произвольные `text-[Npx]` вместо именованных классов | low |

## 3. Решения

- Исправить `sage` в config на `#5C7A59`
- Добавить все необходимые shadow-токены в config
- Заменить все inline styles на Tailwind

## 5. Открытые вопросы

- `TelegramAuthPage` — использует легаси `var(--text2)`, не на Tailwind

## 6. Договорённости

- Все цвета — только через токены (`text-text`, `bg-accent`, `text-text-3` и т.д.). Никаких `#HEX` в JSX.
- Все тени — только `shadow-card`, `shadow-sm`, `shadow-md`, `shadow-accent`, `shadow-sage`, `shadow-top`. Никаких inline boxShadow.
- `EmptyState` — использовать компонент, не писать кастомные пустые состояния.
- `Loader` — для загрузки страниц, `SkeletonCard` — для списков блюд.
