# Dishes — аудит

## 1. Текущее состояние

Рецепты (Dish) — основная сущность. Везде в коде и UI называются "блюда".

**Видимость (Dish.visibility):**
- `PRIVATE` — только автор
- `PUBLIC` — все (в том числе гости)
- `FAMILY` — участники семейной группы (по `groupId`)
- `ALL_GROUPS` — все соучастники любых групп автора

**Основные поля:** `authorId`, `visibility`, `groupId`, `mealTime[]`, `ingredients[]`, `tags[]`, `protein/fat/carbs` (КБЖУ).

**Комментарии:** модель `Comment`. Доступны если пользователь видит блюдо в контексте группы: автор всегда, PRIVATE/PUBLIC — только автор, FAMILY — участники семейной группы, ALL_GROUPS — участники любой общей группы с автором, блюдо в REGULAR-группе — участники этой группы.

**Поиск:** по названию, описанию, ингредиентам, тегам (умный поиск).

**Фильтрация по холодильнику:** два режима — инклюзивный (хотя бы 1 ингредиент) и эксклюзивный (все ингредиенты есть). `toTaste=true` и `isBasic=true` не считаются обязательными.

## 2. Найденные проблемы

| Проблема | Приоритет |
|---|---|
| Старое поле `isPublic Boolean` — не удалено после миграции на visibility | high |
| Системные блюда (authorId=null) не имели visibility=PUBLIC | high |
| Название "Рецепты" в части UI вместо "Блюда" | medium |
| DishFormPage содержала весь код пикера ингредиентов (75 строк внутри) | medium |

## 3. Решения

- `isPublic` — удалён из схемы, везде `visibility`
- Системные блюда обновлены → `visibility=PUBLIC` через seed/скрипт
- Весь UI переименован: "Рецепты" → "Блюда"
- `DishIngredientPicker` вынесен в `/components/domain`

## 4. Реализация

**Файлы:**
- `backend/src/routes/dishes.js` — CRUD, фильтрация, поиск
- `backend/src/routes/comments.js` — CRUD + pin
- `frontend/src/pages/DishFormPage.jsx` — форма создания/редактирования
- `frontend/src/pages/DishDetailPage.jsx` — просмотр, комментарии
- `frontend/src/pages/DishesPage.jsx` — список с поиском и фильтрами
- `frontend/src/components/domain/DishCard.jsx` — карточка блюда (единственная)
- `frontend/src/components/domain/DishIngredientPicker.jsx` — пикер ингредиентов

**Пагинация:** `GET /api/dishes?limit=&offset=` возвращает `{ dishes, total }`.

## 5. Открытые вопросы

- Список покупок: автогенерация из выбранных блюд (в бэклоге)
- Редактирование блюда другим участником FAMILY (сейчас только автор)

## 6. Договорённости

- Сущность — **"блюдо"** везде: в коде, UI, комментариях. Не "рецепт", не "recipe".
- `DishCard` — единственная карточка, альтернативы не создавать.
- Новый экран со списком блюд — всегда через пагинацию (`limit/offset`).
- `toTaste` и `isBasic` ингредиенты не требуют наличия в холодильнике — не нарушать эту логику.
