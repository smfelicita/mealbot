### Поведение ИИ для гостя
- Лимит: 5 сообщений в день (по IP или сессии)
- Нет контекста холодильника
- При попытке спросить про холодильник → предложить зарегистрироваться
- При клике на "Холодильник" → модальное окно с предложением регистрации

## Рецепты и блюда

### Структура рецепта
- Название (рус + англ)
- Описание
- Категория (завтрак / обед / ужин / перекус / суп / салат / десерт / напиток)
- Теги (вегетарианское, быстро, без глютена и т.д.)
- Время приготовления (минуты)
- Сложность (легко / средне / сложно)
- Калории
- **Ингредиенты с количеством** (уже есть в DishIngredient.amount)
- **Пошаговый рецепт** (markdown текст)
- **Фото** — загрузка файла (хранить в облаке S3/Cloudflare R2)
- **Видео** — загрузка файла или ссылка YouTube
- Автор (null = системное блюдо)
- Приватность (видит автор → группа → все)

### Видимость рецептов
- **Системные блюда** (20 штук в базе) — публичные, видны всем
- **Рецепты пользователя** — 4 уровня: PRIVATE / PUBLIC / FAMILY / ALL_GROUPS
- PRIVATE — только автор
- PUBLIC — все пользователи
- FAMILY — только участники семейной группы автора
- ALL_GROUPS — участники всех групп автора одновременно

### Валидация формы рецепта
- Обязательные поля: название, категория
- Показывать ошибки под полями при попытке сохранить без заполнения

### Галерея фото
- До 10 фотографий на рецепт
- Выбор одного основного фото (отображается в карточке и шапке)
- Остальные фото в галерее на странице рецепта

### Рекомендации в карточке рецепта
**Для неавторизованного пользователя** (после описания):
- Блок "Похожие рецепты" — по совпадению ингредиентов

**Для авторизованного пользователя** (после описания, три отдельных блока):
1. **Из вашего холодильника** — рецепты, где все ингредиенты есть в холодильнике
2. **Почти из холодильника** — рецепты с 1–3 недостающими ингредиентами (показывать что купить)
3. **Похожие рецепты** — по совпадению ингредиентов без учёта холодильника

## Холодильник

- Добавление продуктов вручную из списка
- Удаление продуктов
- Дата добавления, срок годности (опционально)
- Режим "из холодильника" — фильтр блюд только из имеющихся продуктов (все ингредиенты должны быть в наличии)
- Семейный холодильник (FAMILY группа) — общий для всех участников
- **На перспективу:** опция "Учитывать точное количество ингредиентов" — учёт amount при фильтрации
- **На перспективу (Pro):** добавление через фото чека

## Авторизация / Регистрация

- Текущая: по email + пароль (JWT)
- **Планируется:**
  - По номеру телефона + подтверждение кодом (SMS)
  - По email + подтверждение кодом (без пароля)
  - Через Google (OAuth / Gmail)

## Группы

- Типы: FAMILY (1 на пользователя, до 10 участников) и REGULAR (до 2, до 1000 участников)
- FAMILY: общий холодильник + общие рецепты
- REGULAR: только общие рецепты
- Роли в группе: OWNER / MEMBER
- Приглашение по коду (groupId)
- Автомиграция личного холодильника → семейный при вступлении в FAMILY

## ИИ-помощник

- Чат с Claude API
- Контекст: продукты в холодильнике, доступные блюда
- Быстрые подсказки (кнопки)
- История чата сохраняется (для авторизованных)
- Для гостей: лимит 5 сообщений, история не сохраняется

---

## Медиафайлы

- Загрузка фото и видео через форму
- Хранение: Cloudflare R2 (дешёвый S3-совместимый) или Supabase Storage
- Поддержка форматов: JPEG, PNG, WebP для фото; MP4 для видео
- Ограничение размера: фото до 5MB, видео до 100MB

- Управление холодильником

## Этапы реализации

- Добавить `role` в User (USER / PRO / ADMIN)
- Добавить `subscriptionUntil` в User

- Миграция БД

- Настроить Cloudflare R2 или Supabase Storage

- API CRUD для рецептов пользователя
- Страница "Мои рецепты"
